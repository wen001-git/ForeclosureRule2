贷款状态图 — 4 个画法对比（决策用）

目的：FCL / LM / BK 是叠加在贷款上的独立并发维度，不是逾期轴上的「下一个状态」。下面给出你现有的线性图 + 3 个候选画法，供对比选择。本页是临时决策稿（不改动 fcl_pipeline.html 与 docs）。

方案 A — 正交并行区（4 维同时演进）不推荐（见下）

优点
  • 形式最正确：一眼看出 4 维各自独立、可并发
  • 彻底消除「FCL/LM/BK 是逾期下一步」的误读
  • 4 个并排子图、各带维度标题，语法稳
缺点
  • 画不出维度耦合：BK 自动中止令 / LM 审核会把 FCL进行中→FCL-Hold（B 维度内由 C/D 触发的子态），正交独立轴表达不了
  • 缺 P / REO 终态（其实是 A 轴 delinq 的处置取值）——说明 4 维并非完全正交
  • 比线性图「重」，需解释「并行区=同时成立」
flowchart TB subgraph A["A · Delinquency 逾期程度 — 唯一的进展轴"] direction LR a1["Current"] --> a2["D30"] --> a3["D60"] --> a4["D90"] --> a5["D120P"] end subgraph B["B · FCL 止赎 — 独立开关"] direction LR b1["N 未启动"] -->|启动止赎| b2["Y 止赎进行中"] b2 -->|完成 / 撤销| b1 end subgraph C["C · LM 损失缓解 — 独立开关"] direction LR c1["N 无 LM"] -->|开案| c2["Y 在谈方案"] c2 -->|关案 / 失败| c1 end subgraph D["D · BK 破产 — 独立开关"] direction LR d1["N 未破产"] -->|申请破产| d2["Y 自动中止令"] d2 -->|驳回 / discharge / Ch.13| d1 end A ~~~ B B ~~~ C C ~~~ D

读法：贷款在每个区里各处于一个值；4 个值的组合 = 业务状态（如 A=D120P、B=Y、C=N、D=Y）。BK/LM 翻成 Y 时不改变 A 区的值。

方案 B — 主轴 + 标志泳道（时间线/Gantt 风）最直观

优点
  • 最贴近运营直觉：一眼看出某段时间内哪些维度同时为 Y
  • 「重叠区=并发」非常直白
缺点

方案 C — 线性处置链 + 独立布尔标志（改动最小）最小改动

优点
  • 保留你熟悉的主链，仅把 LM/BK/Hold 从链上「摘下来」当标志
  • 改动小、风险低
缺点
  • 并发性仍靠文字/表，不如 A/B 直观

★ 推荐落地方案 — 原线性图为主 + 叠加态虚线标注 + 补充说明推荐

你说得对：方案 A 的「完全正交」假设过度简化——画不出 FCL⇄FCL-Hold 的耦合，也没有 P/REO 终态。而原线性图本就表达了这些，只需把三个「并发叠加态」用虚线区分、并补充说明即可。下图：实线 = delinq 主轴真实取值（Current→D30…→FCL→REO/P）；虚线紫块 = 并发叠加态（LM / BK / FCL-Hold，不改变 delinq）。

配套三条关键澄清（补充说明）：
  • FCL-Hold 是 FCL 的子态，由 LM 审核 / BK 自动中止令触发（非独立第 5 态）：FCL进行中 ⇄ FCL-Hold。
  • LM / BK / Hold 生效期间 delinq 不变（仍 FCL 或原逾期档）——它们是叠加标志，不是逾期轴上的下一步（代码 CREATE_FCL_RELATE_ATTR：Foreclosure / Perf·Non-Perf BK → FCL）。
  • P / REO 是处置终态（A 轴 delinq 的取值），由 FCL/REO 或全额还清到达；Ch.7 discharge ≠ 还清（Lien 存续），故无 BK→P 直达。
